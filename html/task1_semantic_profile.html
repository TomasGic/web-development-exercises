About learning

Do you ever get discouraged in your journey to become a software developer when AI is constantly getting better at writing code?
It's hard to predict what will happen in the long-term but in the near to mid-term I do not see AI as a direct competitor to human software developers. Currently AI cannot think and reason autonomously about complex real-world problems, which means that there are still human software developers needed. AI is simply a tool that software developers can use to increase their productivity.
Is Ruby a difficult programming language to learn?
Compared to other programming languages, especially those that are statically typed, such as C or Java, Ruby is relatively easy to learn because of its elegant syntax which reads almost like English language. Consider this piece of code which iterates through an array and capitalizes the string element. 
                    
                        orgs = ["imf", "who", "ing"]
                        orgs.each { |org| org.capitalize }
If we wanted to write the same script in C, it would look like this: 
                    
                        #include <stdio.h>
                        #include <ctype.h>

                        int main() {    
                            char *orgs[] = {"imf", "who", "ing"};
                            int length = 3;

                            for (int i = 0; i < length; i++) {
                                orgs[i][0] = toupper(orgs[i][0]);
                            }
                            return 0;
                        }
If you are interested to learn more about Ruby, the best place to start is the official documentation page.
Do you prefer to learn from online resources or books?
I like to learn from both. Online resources have the advantage of being more likely up to date, whereas the material in books might often be outdated as the tech is constantly evolving at a fast pace. On the other hand, I find it easier to read and focus on paper than on the screen.
