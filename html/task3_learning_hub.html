About Me

My programming journey began in April 2020when the whole world shut down due to the covid-19 pandemic. I started working from home and had suddenly a lot more time on my hands, so I wanted to use it in a productive manner. In my professional role as a content editor, I was exposed to many repetitive, tedious tasks that I wanted to automate, but I didn't have the skillset to do it. I started learning Python and soon stumbled upon HTML, CSS and Javascript. I started to apply my new knowledge of these tools in my job and gradually I became the technical automation expert in our content department.

By June 2021I fully or partially automated more than 30 workflows that were previously carried out manually and saved our content department more than 1000 hours of work per year.

In November 2025my company announced a round of layoffs and offered my team an opportunity to leave voluntarily. I thought this was a good opporunity to fully transition into software development. In Januar 2026I started a mentoring program with a goal to become a full-stack software developer specialising in Ruby and Ruby on Rails. I decided to make this website as a learning project but also to document my mentoring journey. You can find more of my projects on my Github page.

Recommended resources

Documentation

  • Ruby-Doc: The official API documentation for the Ruby language.
  • Rspec-Doc: Comprehensive guides for the popular Ruby testing framework.
  • MDN Web Docs: Essential for HTML, CSS, and JS reference.
  • Design Patterns in Ruby: Comprehensive guide to design patterns in Ruby.

Books

  • Design Patterns in Ruby by Russ Olsen (2007) - So far the best book on design patterns in general that I have found, comprehensive with easy explanations and practical real-world code examples.
  • A Common-Sense Guide to Data Structures and Algorithms by Jay Wengrow (2020)
    • - One of the best books on data structures and algorithms out there. Contains easy, no-nonsense explanations to complex concepts, every chapter has a handful of exercises along with their solutions.
  • Clean Code: A Handbook of Agile Software Craftsmanship by Robert C. Martin (2009) - This book is a classic in the field of software engineering. Even though it contains code examples in Java, the concepts are applicable to any programming language. I believe it is a must-read for any software developer who is serious about writing clean, professional code.

Reflections & Notes

What I've learned so far

When I started the mentorhsip I already had some knowledge of programming but I wasn't programming consistently. The most important benefit of the mentorship was the move from occasionaly writing automation scripts to writing OOP programs that simulated real domain systems almost every day. With consistency my progress has skyrocketed.

I have become more confident in writing code from scratch without the need to reference documentation or AI models. I have learnt to apply the correct design patterns and I understood how Ruby modules and classes interact with each other.

Biggest Challenges

  1. Eye fatique from spending more time in front of the screen than previously
    I overcame this challenge by taking more frequent breaks and taking supplements for eye health.
  2. Back and neck pain from prolonged sitting and typing
    I overcame this challenge by making my desk setup more ergonomic (getting a smaller keyboard, remapping frequently used keys on the keyboard to better suit hand placement, buying a standing desk to alternate between sitting and standing).
  3. Occasionally having thoughts of self-doubt
    It is natural to occasionally doubt one's abilities, but it's important for these thoughts to just be short-lived and not lower your overal self-confidence. When I get a thought like this I remind myself that I'm still only at the beginning of the journey and that it takes time to become really good at something.
